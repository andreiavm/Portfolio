Processing Generative Art

my role: programming

project type: university assignment

In one of our university courses, we were thought how to program computers by using creative coding as a method of teaching. These generative art pieces are the result of my work after teaching myself processing for just a few days.

moire pattern:

For the first assignment, I coded a moire pattern by using just two simple for loops. The lines loop starting from each corner of the screen and meet up at the x and y position of the cursor.

The number of lines can be incremented as much as the user wishes, and with the number of lines increasing, the illusion becomes less obvious.

what I've learned:

Although this code is relatively simple, I found that generative art brings out a lot of creative potential and it was good practice to practice my programming skills in this creative manner.

generative visuals:

For my second assignment, I have decided to create something more interesting using music, so I built some visuals that could be projected on the walls of an electronic music event, for example.

This idea came from my past experience in event planning, where I have learned many things about design, managment, and organisation.

I used geometry to create for loops with arcs, and the movement is created by incrementing the sw variable which sets the stroke width of the arcs. This animation uses two mathematical functions to generate the arcs in such a pattern.

During this project, I have learned many things about programming, more specifically how to debug my code and how to properly search online for solutions to coding problems.

when aliens attack:

As my final project, I went with building a simple game that is inspired by the space invaders arcade nostalgia.

First, I created some variables to which I assigned the x position of the player, the y and x position of the bullet, and the x and y position of the enemy. I also have an integer variable which is set to 0, and every time the bullet hits the enemy, it increments by 1.

When building this game, I have faced multiple challanges, and the hardest part for me was to fix the collision points between the different elements of the game.

I consider that by building this game I have gained valuable experience in implementing logic through my code, and I learned even more about debugging my code.
